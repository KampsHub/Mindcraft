# Product — PRDs, plans, reviews, feedback, design

Loaded when working on product specs, priorities, feedback triage, or design artifacts.

## Files in this folder

### `prd/`
- **`PRD-CURRENT-STATE.md`** — product spec snapshot. Three 30-day programs (Parachute/Jetstream/Basecamp) + Enneagram add-on. Pricing, feature matrix, current state vs roadmap.

### `plans/`
- **`TODO-REMAINING.md`** — active action tracker. Three sections: what Stefanie needs to do, what Claude needs to do, decisions still needed. Append here; never let it drift.
- **`EXERCISE-AUDIT-PLAN.md`** — plan to audit all 362 exercises for complete Learn → Recognize → Practice arc.

### `reviews/`
- **`MULTI-LENS-REVIEW.md`** — product review through four lenses (learning, emotion, accessibility, business).
- **`qualitative-review.md`** — weekly analytics/qualitative review dashboard template.

### `feedback/`
- **`testing-feedback-notes.md`** — beta testing action log from Google Doc. Status per item (✅ fixed, 💬 question, ⏸️ awaiting decision, 🚫 out of scope).

### `design/`
- **`today-flow/`** — split design spec for the Today landing page
- **`Mindcraft-Intake-Onboarding-Questions.docx`** — intake form questions by program
- **`PRD-Post-Login-Product.docx`** — post-login features spec

## Rules

- **TODO-REMAINING is the single source of truth for active work.** When a task completes, update it in the same session. When a new task surfaces, add it immediately.
- **PRD-CURRENT-STATE is the specification**, not the roadmap. Roadmap aspirations live in TODO-REMAINING or in plans.
- **Testing feedback items** get a status tag when triaged: ✅ / 💬 / ⏸️ / 🚫. Never leave an item unlabeled.
